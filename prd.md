# WinnerLens PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 사업명
**WinnerLens** (가칭)  
쿠팡 셀러를 위한 AI 상품 발굴 & 상품 등록 자동화 솔루션

### 1.2 한 줄 소개 (가치 제안)
쿠팡 판매자가 잘 팔릴 가능성이 높은 상품을 찾고, 선택한 후보를 쿠팡 등록 규격에 맞춰 AI가 자동으로 리스팅 생성·등록해주는 운영 도구.

### 1.3 프로젝트 비전
WinnerLens는 단순한 상품 등록 도구를 넘어, 쿠팡 셀러의 전체 운영 프로세스를 지능화하는 플랫폼으로 성장합니다. 데이터 기반 의사결정과 자동화를 통해 셀러가 전략과 성장에 집중할 수 있도록 지원합니다.

---

## 2. 문제 정의 (Problem Statement)

### 2.1 핵심 문제점
쿠팡 셀러들이 직면한 주요 페인 포인트:

1. **상품 리서치의 비효율성**
   - 신규 상품 발굴 시 트렌드, 경쟁 상황, 마진 분석에 과도한 시간 소요
   - 감에 의존한 상품 선택으로 인한 높은 실패율
   - 데이터 기반 의사결정 도구 부재

2. **복잡한 상품 등록 프로세스**
   - 쿠팡 등록 시 필수 항목이 많아 누락/오류 발생 빈번
   - 정책 위반으로 인한 반려 및 재작업 시간 낭비
   - 대량 등록 시 반복 작업의 피로도

3. **아이템 위너 경쟁의 복잡성**
   - 가격, 배송, 운영 품질 등 복합적 요소 고려 필요
   - 시장 변화에 대한 빠른 대응 어려움
   - 경쟁 상품 분석 및 포지셔닝 전략 수립의 어려움

### 2.2 타깃 페르소나

**페르소나 1: 신규 셀러 (스타트업 단계)**
- 규모: 1~5인
- 특징: 쿠팡 Wing 초기 사용자, 상품 소싱 경험 부족
- 니즈: 무엇을 팔아야 할지 모르겠음, 등록 과정이 복잡함

**페르소나 2: 성장 중인 리셀러**
- 규모: 5~20인
- 특징: 다양한 카테고리 운영, 대량 등록 필요
- 니즈: 등록 자동화, 운영 효율화, 데이터 기반 상품 선정

**페르소나 3: 중형 도매/브랜드 셀러**
- 규모: 20~50인
- 특징: 체계적 운영, 팀 협업 필요
- 니즈: 팀 계정 관리, 고급 분석, 운영 자동화

## 3. 솔루션 (Solution)

### 3.1 핵심 접근 방식
WinnerLens는 **두 가지 핵심 엔진**으로 셀러의 문제를 해결합니다:

#### 1) 상품 발굴 엔진 (Product Discovery Engine)
- 트렌드/수요 신호 분석
- 경쟁/마진/운영 가능성 점수화
- 데이터 기반 후보 상품 추천

#### 2) AI 리스팅 빌더 (AI Listing Builder)
- 상품명, 옵션, 고시정보, 설명 템플릿 자동 생성
- 정책 및 필수항목 자동 검수
- 쿠팡 등록 자동화

### 3.2 차별화 포인트

| 기존 솔루션 | WinnerLens |
|------------|-----------|
| 단순 등록 대행 | 상품 발굴부터 등록까지 end-to-end |
| 수동 검수 | AI 기반 자동 검수 및 가드레일 |
| 일회성 도구 | 지속적 운영 자동화 플랫폼 |
| 경험 의존 | 데이터 기반 의사결정 |

## 4. 핵심 기능 (Core Features)

### 4.1 MVP (Phase 1) 기능

#### F1. 트렌드 기반 후보 추천
**목적**: 시장 트렌드를 분석하여 판매 가능성 높은 상품 후보 제시

**기능 상세**:
- 상승 키워드 분석 (검색량, 증가율)
- 핫 카테고리 탐지
- 시즌/이벤트 기반 추천
- 후보 리스트 자동 생성

**데이터 소스**:
- 쿠팡 검색 트렌드
- 카테고리별 판매 순위
- 외부 트렌드 데이터 (Google Trends, 네이버 쇼핑 등)

**출력**:
- 추천 상품 리스트 (상품명, 카테고리, 예상 수요)
- 트렌드 점수 (1-100)

---

#### F2. 판매 가능성 스코어링
**목적**: 다차원 분석을 통해 상품의 성공 가능성 정량화

**스코어링 요소**:

1. **수요 점수 (Demand Score)** - 30%
   - 검색량 트렌드
   - 카테고리 성장률
   - 시즌성 지수

2. **경쟁 점수 (Competition Score)** - 25%
   - 경쟁 상품 수
   - 가격 경쟁력
   - 리뷰 수/평점 분포
   - 로켓배송 비율

3. **마진 점수 (Margin Score)** - 25%
   - 예상 판매가 vs 원가
   - 수수료/배송비 고려
   - 마진율 계산

4. **운영 가능성 점수 (Operability Score)** - 20%
   - 재고 관리 난이도
   - 배송 복잡도
   - 반품률 예측
   - 규정 준수 난이도

**출력**:
- 종합 점수 (1-100)
- 각 요소별 세부 점수
- 개선 제안 사항

---

#### F3. AI 상품 정보 생성
**목적**: 쿠팡 등록에 필요한 모든 정보를 AI가 자동 생성

**생성 항목**:

1. **상품명 생성**
   - SEO 최적화된 상품명
   - 키워드 포함
   - 쿠팡 정책 준수 (금지어 제외)

2. **상품 속성/옵션**
   - 색상, 사이즈 등 옵션 자동 추출
   - 옵션별 가격/재고 템플릿

3. **상품 고시정보**
   - 카테고리별 필수 고시정보 자동 채움
   - KC 인증, 제조사 정보 등

4. **상품 설명**
   - 구조화된 상품 설명 템플릿
   - 주요 특징/스펙 정리
   - 이미지 배치 가이드

**AI 모델**:
- GPT-4 기반 텍스트 생성
- 쿠팡 정책 학습 모델
- 카테고리별 특화 프롬프트

---

#### F4. 자동 검수 (가드레일)
**목적**: 등록 전 오류/위반 사항 자동 탐지 및 수정

**검수 항목**:

1. **필수 항목 체크**
   - 모든 필수 필드 입력 확인
   - 이미지 규격 확인
   - 옵션 완성도 검증

2. **정책 준수 검증**
   - 금지 표현 탐지 (과대광고, 의료 효능 등)
   - 오인 소지 문구 체크
   - 저작권 위반 가능성 검토

3. **품질 검증**
   - 상품명 길이/형식
   - 이미지 해상도/품질
   - 설명 완성도

**출력**:
- 검수 통과/실패 상태
- 오류 항목 리스트
- 수정 제안 사항

---

#### F5. 쿠팡 등록/수정 자동화
**목적**: 검수 완료된 상품을 쿠팡에 자동 등록

**기능**:
- 쿠팡 OPEN API 연동
- 단일/대량 등록 지원
- 등록 상태 모니터링
- 오류 처리 및 재시도

**워크플로우**:
```
1. 상품 정보 최종 확인
2. API 호출 (상품 등록)
3. 등록 결과 수신
4. 성공/실패 로깅
5. 실패 시 원인 분석 및 재시도
```

---

## 5. 사용자 플로우 (User Flow)

### 5.1 메인 플로우: 상품 발굴 → 등록

```
[시작]
  ↓
[1. 대시보드 접속]
  ↓
[2. 트렌드 추천 확인]
  - 카테고리 선택
  - 필터 적용 (마진, 경쟁 등)
  ↓
[3. 후보 상품 리스트 조회]
  - 스코어 기준 정렬
  - 상세 분석 보기
  ↓
[4. 상품 선택]
  - 단일 또는 복수 선택
  ↓
[5. AI 정보 생성]
  - 자동 생성 실행
  - 생성 결과 확인
  ↓
[6. 정보 편집 (선택)]
  - 상품명, 설명 수정
  - 옵션 조정
  ↓
[7. 자동 검수]
  - 검수 실행
  - 오류 확인 및 수정
  ↓
[8. 등록 실행]
  - 쿠팡 API 호출
  - 등록 결과 확인
  ↓
[9. 등록 완료]
  - 성공 알림
  - 다음 상품 진행 또는 종료
```

### 5.2 서브 플로우

**플로우 A: 수동 상품 등록**
- 사용자가 직접 상품 정보 입력
- AI 보조 기능 활용 (자동완성, 제안)
- 검수 및 등록

**플로우 B: 대량 등록**
- CSV/Excel 파일 업로드
- 일괄 AI 생성
- 일괄 검수 및 등록

**플로우 C: 등록 상품 수정**
- 등록된 상품 조회
- 정보 수정
- 재등록 또는 업데이트

## 6. 기술 스택 (Technology Stack)

### 6.1 프론트엔드
- **Framework**: React.js / Next.js
- **UI Library**: Material-UI 또는 Ant Design
- **State Management**: Redux 또는 Zustand
- **Styling**: Tailwind CSS

### 6.2 백엔드
- **Framework**: Node.js (Express) 또는 Python (FastAPI)
- **Database**: PostgreSQL (관계형), Redis (캐싱)
- **API**: RESTful API + GraphQL (선택)
- **인증**: JWT, OAuth 2.0

### 6.3 AI/ML
- **LLM**: OpenAI GPT-4 API
- **데이터 분석**: Python (Pandas, NumPy)
- **스코어링 모델**: 자체 개발 알고리즘 + ML 모델

### 6.4 외부 연동
- **쿠팡 OPEN API**: 상품 등록/수정/조회
- **트렌드 데이터**: Google Trends API, 네이버 쇼핑 API
- **크롤링**: Puppeteer, Scrapy (정책 준수 범위 내)

### 6.5 인프라
- **클라우드**: AWS 또는 GCP
- **컨테이너**: Docker, Kubernetes
- **CI/CD**: GitHub Actions, Jenkins
- **모니터링**: Sentry, DataDog

## 9. 출시 전략 (Go-to-Market Strategy)

- 특정 카테고리 집중 (예: 생활용품, 뷰티)

## 12. 준수 사항 및 원칙 (Compliance & Principles)

### 12.1 운영 원칙

WinnerLens는 다음 행위를 **절대 지원하지 않습니다**:

❌ **금지 사항**:
- 중복 등록 / 노출 조작
- 허위 구매 / 리뷰 조작
- 정책 위반 유도
- 타 셀러 데이터 무단 수집
- 과대/허위 광고 생성

✅ **지향점**:
- 데이터 기반 합리적 추천
- 정책 준수 자동화
- 투명한 운영 지원
- 셀러 교육 및 성장 지원

### 12.2 면책 조항

- WinnerLens는 "아이템 위너 보장"을 약속하지 않습니다
- 제공하는 것은 **가능성 추천 + 운영 최적화 도구**입니다
- 최종 의사결정 및 책임은 셀러에게 있습니다
